## Суть метода

Метод заключается в использовании двух (или более) указателей для обхода структуры данных. Вместо вложенных циклов (O(n²)), мы проходим данные за один проход (O(n)), перемещая указатели по определенным правилам.

## Ключевые паттерны для распознавания

1. **"Найти пару"** → два указателя с разных сторон
    
2. **"Палиндром"** → два указателя с разных сторон (частный случай поиска пары)
    
3. **"Удалить дубликаты"** → два указателя с одной стороны
    
4. **"Цикл/середина списка"** → быстрый и медленный
    
5. **"Слияние массивов"** → два указателя на разных массивах

### 🎯 "Найти пару" → два указателя с разных сторон

**Суть подхода:**

- Массив отсортирован (или может быть отсортирован без потери смысла задачи)

### 🎯 "Удалить дубликаты" → два указателя с одной стороны

**Суть подхода:**  
Оба указателя стартуют с начала и движутся в одном направлении. "Медленный" указатель фиксирует позицию записи, "быстрый" — читает новые элементы. Когда быстрый встречает элемент, который должен быть сохранен, он копируется в позицию медленного, и медленный сдвигается.

**Когда применять:**

- Массив отсортирован (дубликаты идут подряд)
    
- Требуется модифицировать массив "на месте" (in-place)
    
- Нужно сохранить относительный порядок элементов

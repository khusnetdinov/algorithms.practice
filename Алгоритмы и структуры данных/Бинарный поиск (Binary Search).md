# Документация: Решение задач методом Бинарного поиска

## Суть метода

Бинарный (двоичный) поиск — это алгоритм поиска элемента в **отсортированном** массиве (или множестве со свойством монотонности). Вместо линейного просмотра каждого элемента, алгоритм работает, сокращая пространство поиска вдвое на каждом шаге.

Ключевая идея: на каждом этапе мы сравниваем искомое значение со средним элементом текущего диапазона. На основе сравнения мы отбрасываем ту половину массива, в которой искомого элемента гарантированно нет.

**Базовый принцип работы (итеративно или рекурсивно):**
1. Определяем границы поиска (левый `L` и правый `R` индексы).
2. Пока границы не сомкнулись (`L <= R`), находим середину `mid = L + (R - L) / 2`.
3. Сравниваем `nums[mid]` с целью (`target`).
   - Если `nums[mid] == target`: **нашли** (базовый случай).
   - Если `nums[mid] < target`: значит цель справа, смещаем левую границу `L = mid + 1`.
   - Если `nums[mid] > target`: значит цель слева, смещаем правую границу `R = mid - 1`.

**Обязательные условия применимости:**
1. **Монотонность:** Данные должны быть отсортированы (по возрастанию или убыванию) либо обладать свойством, которое можно представить как монотонную функцию (например, "функция ошибки").
2. **Произвольный доступ:** Мы должны иметь возможность быстро обратиться к любому элементу по индексу (массивы подходят идеально, связные списки — нет).

## Ключевые паттерны для распознавания

1. **"Поиск в отсортированном массиве"** → Самая очевидная группа. Массив явно отсортирован, нужно найти число, позицию или проверить наличие.
2. **"Поиск границы (First/Last Position)"** → Ищем не любое вхождение, а самое левое (первое) или самое правое (последнее). Требует модификации условия, чтобы не останавливаться на первом найденном.
3. **"Поиск в почти отсортированном массиве"** → Массив изначально отсортирован, но затем сдвинут (ротирован).
4. **"Поиск ответа (Binary Search on Answer)"** → Самый мощный и неочевидный паттерн. Мы не ищем элемент в массиве, а **подбираем ответ** (число), проверяя, выполняется ли некоторое условие. Если для числа `X` условие выполняется, а для `X-1` нет — мы нашли ответ. Классические задачи: "найти минимальное значение, при котором что-то работает", "задачи с дробями".
5. **"Поиск пика (Peak)"** → Поиск элемента, который больше своих соседей. Массив не обязательно полностью отсортирован, но имеет определенный паттерн (возрастание, затем убывание).

### 🎯 "Поиск ответа (Binary Search on Answer)" — подбор значения

**Суть подхода:** Это высший пилотаж. Задача звучит как оптимизационная ("минимизировать максимум", "найти наибольшее... при условии..."). Мы не можем просто взять и посчитать ответ. Мы **угадываем** ответ (потенциальное число) с помощью бинарного поиска, а затем проверяем с помощью вспомогательной функции (часто жадной), возможно ли это.

**Когда применять:**
- Если сказано "наименьшее максимальное значение" (min of max) или "наибольшее минимальное" (max of min).
- Если задачу можно переформулировать как: "Является ли число X допустимым решением?" (Yes/No question), причем для всех чисел меньше X ответ `False`, а для всех чисел больше X — `True` (или наоборот).

### 💡 Рекомендуемый порядок прохождения

1. **Начните с классики** (704, 374) — поймите механизм сужения границ и работу с индексами (`L`, `R`, `mid`).
2. **Перейдите к поиску границ** (278, 35, 34) — научитесь искать первое вхождение и обрабатывать краевые условия.
3. **Освойте поиск ответа на простых числах** (69, 367, 441) — привыкните к идее, что мы ищем не элемент, а число, удовлетворяющее условию.
4. **Решайте задачи на ротированные массивы** (153, 33) — это прокачает понимание инвариантов (какая часть отсортирована).
5. **Разберитесь с поиском пика** (162, 852) — поймите, как двигаться по направлению градиента.
6. **Переходите к главному — поиску ответа на сложных условиях** (1011, 875, 1283). Это самый важный этап.
7. **Завершите сложными задачами** (410, 4, 719) — для полного понимания глубины метода.
